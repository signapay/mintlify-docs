---
title: "Clients Overview"
description: "Manage clients across the SignaPay hierarchy"
---

Clients represent every organization node in the SignaPay hierarchy — from the platform operator down through processors, banks, ISOs, sub-ISOs, and agents. Every entity that owns settings, users, or processing relationships is a client record.

## What's included

### Core

| Endpoint | Description |
| --- | --- |
| [List clients](/api-reference/clients/list-clients) | Retrieve a paginated list of clients |
| [Get client](/api-reference/clients/get-client) | Retrieve a single client by ID |
| [Update client](/api-reference/clients/update-client) | Update an existing client record |
| [Update status](/api-reference/clients/update-client-status) | Transition a client to a new status (activate, deactivate, close) |
| [Search by identifier](/api-reference/clients/search-clients) | Look up a client by MID, DBA, or other identifier |
| [Get defaults](/api-reference/clients/get-client-defaults) | Retrieve default configuration scoped to a client |
| [Update defaults](/api-reference/clients/update-client-defaults) | Update default configuration for a client |
| [Decrypt sensitive data](/api-reference/clients/decrypt-client) | Decrypt a protected field on demand |
