---
title: "Validations Overview"
description: "Run identity, credit, and compliance checks against an application"
---

Validation endpoints trigger third-party identity and compliance checks against an application record. Results are stored on the application and can be retrieved individually. Run a completeness check before submission to ensure all required fields are populated.

| Endpoint | Description |
| --- | --- |
| [Validate completeness](/api-reference/leads/validate-lead-completeness) | Check all required fields before submission |
| [DataX verification](/api-reference/leads/lead-datax-validation) | Run a DataX identity verification |
| [Equifax credit](/api-reference/leads/lead-equifax-credit) | Run an Equifax credit report |
| [Equifax state ID](/api-reference/leads/lead-equifax-state-id) | Run an Equifax state ID verification |
| [GIACT validation](/api-reference/leads/lead-giact-validation) | Run a GIACT bank account validation |
| [MATCH inquiry](/api-reference/leads/lead-match-validation) | Run a MATCH Pro terminated merchant screening |
| [TinCheck](/api-reference/leads/lead-tincheck-validation) | Run a TinCheck tax ID verification |
| [List validations](/api-reference/leads/list-lead-validations) | Retrieve all validation records for an application |
| [Get validation](/api-reference/leads/get-lead-validation) | Retrieve a single validation record |
