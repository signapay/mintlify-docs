## Prerequisites

Before you begin, make sure you have:

- A SignaPay PayHub account with API access enabled
- Either a user login (email \+ password) or a long-lived **API key** issued for your client — see [Authentication](/getting-started/tutorials/authentication/overview) for details
- An HTTP client such as `curl`, Postman, or your preferred SDK

All requests must be made over HTTPS to the base URL:

```text
https://api.signapay.com
```

---

## Step 1 — Obtain a token

PayHub supports two authentication schemes. Use whichever fits your integration:

<Tabs>
  <Tab title="Session token (user login)">
    Exchange an email and password for a short-lived bearer token. If your account has MFA enabled, a second call is required to complete the challenge.

    **1a. Create a session**

    ```bash
    curl -s -X POST https://api.signapay.com/v1/crm/sessions \
      -H "Content-Type: application/json" \
      -d '{"email": "you@example.com", "password": "••••••••"}'
    ```

    A successful response returns a `token` and, if MFA is pending, a `mfaRequired: true` flag.

    **1b. Complete MFA (if required)**

    ```bash
    curl -s -X POST https://api.signapay.com/v1/crm/sessions/mfa \
      -H "Content-Type: application/json" \
      -d '{"code": "123456"}'
    ```

    Once MFA is verified, the response includes your bearer token. Pass it in every subsequent request:

    ```text
    Authorization: Bearer <token>
    ```

    See [Create session](/getting-started/tutorials/authentication/create-session) and [Verify MFA](/getting-started/tutorials/authentication/verify-mfa) for full request and response schemas.
  </Tab>
  <Tab title="API key (service-to-service)">
    For automated integrations, use a long-lived API key instead of logging in on each request. Keys are managed through the [Client API Key](/api-reference/clients/get-client-api-key) endpoints.

    Pass the key in every request:

    ```text
    Authorization: Api-Key <your-api-key>
    ```

    No session creation step is needed.
  </Tab>
</Tabs>

---

## Step 2 — Make your first request

With your token or API key in hand, try fetching the list of agents in your account:

```bash
curl -s https://api.signapay.com/v1/crm/agents \
  -H "Authorization: Bearer <token>"
```

A successful `200` response returns a paginated list of agent records. If the list is empty, your account may not have any agents configured yet — that's fine; the important thing is that the request authenticated successfully.

<Tip>
  Replace `Bearer <token>` with `Api-Key <your-api-key>` if you're using a service key instead.
</Tip>
